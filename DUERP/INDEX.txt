  RÈGLES D'ARTICULATION CLÉS
  ───────────────────────────
  1. REGLEMENTATION.txt (01_FONDATIONS) est le socle commun.
     Chaque module _REGLEMENTATION n'y ajoute que les textes SPÉCIFIQUES
     à son risque (ex : Directive 2002/44/CE pour les vibrations).
     → Jamais répéter L4121-1 dans un module : il est dans le socle.

  2. COTATION.txt est l'unique source de vérité pour les formules.
     Toutes les tables "criticité APR" dans les fichiers METHODE
     sont des déclinaisons de ce système central.
     → Si le système de cotation change, ne modifier que COTATION.txt.

  3. STANDARD_PRESELECTION.txt définit les 3 questions de chaque module.
     Les écrans de présélection dans le SaaS en sont la traduction directe.
     → Ne pas dupliquer ces questions dans les fichiers METHODE.

  4. Chaque paire (REGLEMENTATION + METHODE) est autonome.
     Un module peut être lu indépendamment des autres.
     → Mais METHODE référence toujours REGLEMENTATION pour les seuils légaux.

  5. 00_SOURCES ne doit jamais être modifié.
     Toute mise à jour réglementaire se fait dans les fichiers METHODE/REGLEMENTATION,
     pas dans les sources originales.

  GRANULARITÉ D'ÉVALUATION
  ─────────────────────────
  L'unité d'analyse de base est l'OPÉRATION (pas le poste, pas l'entreprise).

  Entreprise
  └── Poste (ex : soudeur)
      ├── Opération A : soudage à l'arc
      │     ├── Risque Chimique (fumées)      → Module M06
      │     └── Risque Rayonnements (UV)      → Module M09
      ├── Opération B : meulage
      │     ├── Risque Bruit                  → Module M01
      │     └── Risque Vibrations MB          → Module M02
      └── Opération "Toutes opérations"       ← opération transversale
            └── Risque RPS                   → Module M05

  Opération "Toutes opérations" = risques uniformes sur tout le poste
  (bruit ambiant, température, RPS organisationnel).

  CRITICITÉ FINALE DU POSTE = MAX des criticités de toutes ses opérations
  pour un risque donné.

  SYSTÈME DE COTATION
  ────────────────────
  Risque AIGU     : Criticité = Gravité × Probabilité       (échelle 1→16)
  Risque CHRONIQUE: Criticité = Gravité × Durée d'Exposition (échelle 1→16)
  Plan de Maîtrise: Criticité résiduelle = Criticité brute × Coefficient PM
  Coefficient PM  : 0.0 (risque supprimé) → 1.0 (aucune mesure)

  PRÉSÉLECTION PAR MODULE
  ────────────────────────
  Avant chaque module d'analyse : 3 questions binaires OUI/NON.
  0 OUI → Criticité 1 automatique, module ignoré.
  1+ OUI → Module complet déclenché.
  → Détail : 02_METHODOLOGIE/STANDARD_PRESELECTION.txt

  Base de données : Supabase (PostgreSQL + Auth)
  Paiements : Stripe
  Déploiement : Vercel
